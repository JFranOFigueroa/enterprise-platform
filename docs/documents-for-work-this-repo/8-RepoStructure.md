# Repository Structure

## Design Principle

The repository is a product. It must be self-documenting and reproducible.

## Structure

- ADR/ - Architecture Decision Records
- applications/ - Applications consuming the platform
- automation/ - Ansible automation
- bootstrap/ - Platform bootstrap
- docs/ - Documentation
- environments/ - Environment overrides
- infraestructure/ - Cloud-agnostic infrastructure
- platform/ - Shared platform services
- tests/ - Test suites
- tools/ - CLI tools and generators
